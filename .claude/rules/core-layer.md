# Core Layer Rules — lilProject

## Overview

The core layer is the heart of the project, where the business logic and fundamental rules reside. The rules defined in this layer are essential to ensuring the integrity, consistency, and quality of the code. This document establishes the guidelines that must be followed when developing the core layer. This layer acts as the backend of the project, where business rules are implemented and the central system logic is defined. It is responsible for ensuring the system works correctly and meets the established requirements.

The layer is split into two sublayers: **Domain** and **Application**. The Domain layer is responsible for representing the business model, while the Application layer orchestrates operations and interactions between components.

The rules are in the `domain-layer.md` and `application-layer.md` files, respectively. Each file contains the specific rules for each layer, ensuring the code is organized, consistent, and easy to maintain.
